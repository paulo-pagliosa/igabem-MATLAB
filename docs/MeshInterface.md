# MeshInterface — A Brief User Manual

[← Back to README](../README.md)

`MeshInterface` is the graphic tool used to visualize the geometry and analysis results of a BEA/IGA model. It extends `MeshRenderer` (basic mesh rendering) and adds:

- a **toolbar** with navigation/selection tools;
- **mouse interaction** for picking elements (patches) and nodes;
- MATLAB commands (public methods of the object) to control display, selection, colors, vectors, boundary conditions, etc.

## 1. Creating a mesh and opening the interface

A `Mesh` object is created with the `readMesh` function, which reads one or more mesh files generated by the project's C++ preprocessor (as described in [[1]](../README.md/#ref1)):

```matlab
mesh = readMesh('-f', 'tests/tee/tee.be');
mi = MeshInterface(mesh);
```

'readMesh' accepts one or more filenames — with extension '.be' for Bézier elements extracted from a T-spline CAD model — plus optional flags, given as separate string arguments:

| Flag | Meaning |
|---|---|
| `'-f'` | the *next* file has no element face data (default: with face data) |
| `'-g'` | set the glue flag to `true` (default) |
| `'-a'` | set the glue flag to `false` |

When more than one file is given, the resulting meshes are combined:

- if the glue flag is `true` (default), nodes at the same position are merged, producing continuous elements across files;
- if `false`, coincident nodes are replicated, producing semi-discontinuous elements.

```matlab
mesh = readMesh('part1.be', 'part2.be');       % glued (default)
mesh = readMesh('-a', 'part1.be', 'part2.be'); % kept separate
```

Once the mesh is available, `MeshInterface(mesh)` opens the "Mesh View" window, sets up camera, lighting, toolbar and camera gizmo, and draws the mesh.

## 2. The window and toolbar

The window opens with a simple toolbar containing 5 toggle buttons (only one of the first four stays active at a time; the color-bar button is independent):

| Tooltip | Action | Edit mode |
|---|---|---|
| **Select elements** (active by default) | Enables picking elements/nodes with the mouse | `EditMode.SELECT` |
| **Rotate** | Enables 3D camera rotation (`rotate3d`) | `EditMode.ROTATE` |
| **Move** | Enables panning the view | `EditMode.MOVE` |
| **Zoom In** | Enables zooming by drag/click | `EditMode.ZOOM` |
| **Show/Hide Color Bar** (separated from the others) | Shows/hides the color bar of the current color map | — |

The current edit mode can also be changed programmatically:

```matlab
mi.setEditMode(EditMode.ROTATE);
```

`EditMode` has exactly four values: `SELECT`, `ROTATE`, `MOVE`, `ZOOM`.

## 3. Mouse interaction

Mouse behavior depends on the **edit mode** currently active in the toolbar.

### 3.1 Select mode (default)

- **Left click** on an element (patch): selects **only** that element, replacing any previous selection (`SelectMode.SET`).
- **Right click / Ctrl+click** (`SelectionType == 'alt'`) on an element: **adds** the element to the current selection (`SelectMode.ADD`).
- **Shift+click** (`SelectionType == 'extend'`) on an element: **removes** the element from the current selection, if selected (`SelectMode.REMOVE`).

These correspond to the internal modes `SelectMode.SET`, `ADD` and `REMOVE`, also used by the `selectElement`/`selectElements` commands (§5.2).

### 3.2 Node picking (keyboard shortcut `V`)

Pressing **V** while the window has focus toggles a temporary node-picking mode:

- On: patches are hidden and edges/load points/vectors are temporarily turned off; mesh **nodes** become visible and clickable.
- Clicking a node (with `Select` mode active) selects it, equivalent to `mi.selectNode(id)`.
- Pressing **V** again restores the previous visualization state (flags saved before entering node-picking mode).

This shortcut only has effect while the edit mode is `EditMode.SELECT`.

### 3.3 Rotate / Move / Zoom

In `Rotate`, `Move` and `Zoom` modes, mouse interaction is delegated to MATLAB's native tools (`rotate3d`, `pan`, `zoom`), configured to act only on the mesh axes (not on the camera gizmo). After any camera movement, the camera gizmo and the scene lights (`camlight`) are updated automatically.

## 4. Public properties

### 4.1 Editable (read/write)

| Property | Default | Description |
|---|---|---|
| `groundFaceAlpha` | `1` | Opacity (0–1) of the ground plane. |
| `groundFaceColor` | `[0.8 0.8 0.8]` | Fill color of the ground plane. |
| `groundEdgeColor` | `[0.4 0.4 0.4]` | Grid line color of the ground plane. |
| `selectElementColor` | `[0 0.3 0.6]` | Highlight color of selected elements. |
| `nodeElementColor` | `[1 1 0]` | Highlight color of the elements incident to a selected node (see `showNodeElements`). |

```matlab
mi.selectElementColor = [1 0 0]; % selected elements turn red
mi.groundFaceAlpha = 0.5;        % semi-transparent ground
```

### 4.2 Read-only

| Property | Default | Description |
|---|---|---|
| `undeformedMeshAlpha` | `0.2` | Opacity of the undeformed mesh overlay (see `deformMesh`). Set with `setUndeformedMeshAlpha`. |
| `undeformedMeshColor` | `[0.9 0.9 0.9]` | Color of the undeformed mesh overlay. Set with `setUndeformedMeshColor`. |
| `patchEdgeColor` | `[0 0 0]` | Color of patch edges. Set with `setPatchEdgeColor`. |
| `vectorColor` | orange `[255 128 0]/255` | Color of vectors (displacement/traction). Set with `setVectorColor`. |
| `vectorScale` | `3` | Scale factor of vectors. Set with `setVectorScale`. |
| `creaseEdgeWidth` | `2` | Width of crease edges (region boundaries). Set with `setCreaseEdgeWidth`. |
| `bounds` | — | Array of `BoundingBox` objects (axis-aligned bounding boxes, see [`BoundingBox.m`](../graphics/BoundingBox.m)) describing the spatial extent of the tessellated mesh. Computed lazily on first access and cached; the cache is cleared (recomputed on next access) whenever the tessellated geometry changes (e.g. after `remesh`, `moveNode`, `moveElement`, or `deformMesh`). |

**Remark:** a *region* is a set of elements delimited by *crease* edges, see [[1]](../README.md/#ref1).

### 4.3 Dependent

| Property | Description |
|---|---|
| `projection` | Camera projection type of the axes (`'perspective'` or `'orthographic'`), forwarded directly to `axes.Projection`. |

```matlab
mi.projection = 'orthographic';
```

### 4.4 Properties inherited from `MeshRenderer`

`MeshInterface` also exposes the following properties, defined by its base class `MeshRenderer`:

| Property | Default | Access | Description |
|---|---|---|---|
| `nodeProperties` | `PointProperties('blue', 'o', 5)` | read/write | Color, marker shape and size used to draw mesh nodes (e.g. `mi.nodeProperties.color = 'red'`). |
| `vertexProperties` | `PointProperties('cyan', 'o', 2)` | read/write | Color, marker shape and size used to draw patch vertices. |
| `edgeProperties` | `LineProperties('blue', '-', 1)` | read/write | Color, line style and width used to draw patch edges; also settable as a whole through `setEdgeColor` (color only). |
| `faceColor` | `[0.7 0.7 0.7]` | read-only | Base face color of the patches; set through `setFaceColor`. |
| `faceAlpha` | `1` | read-only | Base face opacity (0–1); set through `setFaceAlpha`. |
| `normalColor` | `[102, 0, 102] / 255` | read-only | Color of vertex normal vectors; set through `setNormalColor`. |
| `normalScale` | `2` | read-only | Scale factor of vertex normal vectors; set through `setNormalScale`. |
| `mesh` | — | read-only | The underlying `Mesh` object rendered by this interface. |

`PointProperties(color, shape, size)` and `LineProperties(color, style, width)` are plain property holders (no methods besides their constructor); their fields can be read or written directly, e.g. `mi.edgeProperties.width = 2`.

## 5. Commands (public methods)

### 5.1 Camera and navigation

| Command | Description |
|---|---|
| `setView(azimuth, elevation)` | Sets the camera view angles (degrees). |
| `zoomIn(scale)` | Zooms in by dividing the camera view angle by `scale`. |
| `zoomOut(scale)` | Zooms out by multiplying the camera view angle by `scale`. |
| `showCameraGismo(flag)` | Shows (`true`, default) or hides (`false`) the camera-orientation gizmo. |
| `updateCameraGismo(location)` | Updates the camera gizmo; if `location` is given and differs from the current one, repositions the gizmo. |

```matlab
mi.setView(45, 20);
mi.zoomIn(1.5);
```

### 5.2 Element (patch) selection

| Command | Returns | Description |
|---|---|---|
| `selectElement(i, mode)` | — | Selects element `i`. `mode` is optional (`SelectMode.SET/ADD/REMOVE`; defaults to the current mouse select mode). |
| `selectElements(i)` | — | Adds every element in vector `i` to the current selection. |
| `deselectElement(i)` | — | Removes element `i` from the selection. |
| `deselectAllElements()` | — | Clears the whole element selection. |
| `selectAllElements()` | — | Selects every element of the mesh. |
| `isSelectedElement(i)` | `logical` | Whether element(s) `i` are currently selected. |
| `selectedElements()` | array of `Element` | Returns the currently selected elements. |
| `selectShell(sid)` | — | Selects every element of a shell. Without an argument, uses the shell of the first currently selected element. |
| `pickRegions(eids)` | `eids` | Computes (without changing the selection) the IDs of every element belonging to the regions containing the seed elements `eids` (or the current selection, if omitted). |
| `selectRegions(eids)` | `eids` | Same as `pickRegions`, but also replaces the current selection with the result. |

> **Remark:** a *shell* is a set of boundary elements forming a connected component of a solid's surface. For example, a hollow sphere has two shells, one outer and one inner.

```matlab
mi.selectElement(12);
mi.selectElements([3 4 5]);
els = mi.selectedElements;
mi.selectRegions();         % selects the regions of the current selection
```

### 5.3 Node selection and manipulation

| Command | Returns | Description |
|---|---|---|
| `selectNode(i)` | — | Selects the node of index `i`. |
| `deselectNode()` | — | Clears the current node selection. |
| `selectedNode()` | `Node` | Returns the currently selected node (empty if none). |
| `moveNode(t)` | — | Moves the selected node by displacement `t`, retessellating the affected elements and redrawing. |
| `showNodeElements(flag)` | — | Highlights (`true`) or unhighlights (`false`) the elements incident to the selected node, using `nodeElementColor`. Requires a selected node. |

```matlab
mi.selectNode(7);
mi.showNodeElements(true);
mi.moveNode([0 0 0.1]);
```

### 5.4 Patch and edge visibility

| Command | Description |
|---|---|
| `hidePatches(flag)` | Hides (`true`, default) the currently selected elements, or reveals (`false`) every currently hidden element. |
| `hideAllPatches()` | Selects every element and hides them. |
| `showPatchEdges(flag)` | Shows/hides patch (tessellation) edges. |
| `setPatchEdgeColor(color)` | Sets the color of patch edges. |
| `setCreaseEdgeWidth(width)` | Sets the width of crease edges (minimum `1`). |
| `showEdges(flag)` | Overrides `MeshRenderer.showEdges`: shows/hides mesh element edges (colored with `edgeProperties.color`, inherited from `MeshRenderer`), then reapplies the highlight of the currently selected elements. |
| `setEdgeColor(color)` | Overrides `MeshRenderer.setEdgeColor`: sets `edgeProperties.color` and forces the edges to be shown, then reapplies the selection highlight. |
| `setFaceColor(color)` | Overrides `MeshRenderer.setFaceColor`: sets the base face color (`faceColor`) of every patch, then reapplies the selection highlight on top. |

The following `MeshRenderer` commands are inherited unchanged and remain valid on a `MeshInterface` object:

| Command | Description |
|---|---|
| `showVertices(flag)` | Shows/hides patch vertex markers, colored with `vertexProperties`. |
| `setVertexColor(color)` | Sets `vertexProperties.color` and shows the vertices. |
| `showNodes(flag)` | Shows/hides mesh node markers, colored with `nodeProperties`. |
| `setNodeColor(color)` | Sets `nodeProperties.color` and shows the nodes. |
| `showVertexNormals(flag)` | Shows/hides vertex normal vectors. Passing the string `'update'` instead of a flag forces a redraw of already-visible normals (e.g. after the geometry changes). |
| `setNormalScale(scale)` | Sets the scale factor of vertex normal vectors (`scale < 0` is ignored). |
| `setNormalColor(color)` | Sets the color of vertex normal vectors. |
| `setFaceAlpha(alpha)` | Sets the base face opacity of every patch (clamped to `[0, 1]`). |

```matlab
mi.showNodes(true);
mi.setNodeColor('red');
mi.showVertexNormals(true);
mi.setFaceAlpha(0.6);
```

### 5.5 Colors and color maps (scalar results)

`MeshInterface` uses a `ScalarExtractor` (see [`ScalarExtractor.m`](../vis/ScalarExtractor.m)) to map a scalar field onto the tessellated mesh.

| Command | Description |
|---|---|
| `showColorMap(flag)` | Overrides `MeshRenderer.showColorMap`. When enabling (`true`), recomputes the current scalar field (`scalarExtractor.execute`) and maps each patch's scalar values to interpolated face colors (base behavior, inherited from `MeshRenderer`, using the axes colormap set at construction — `'jet'`). Also disables node-element highlighting, reapplies the element-selection highlight, and rebuilds the color bar if it is visible. When disabling (`false`), reverts every patch to the plain `faceColor`. |
| `updateColorMap()` | Forces the current color map to be recomputed/redrawn. |
| `setScalars(field, ...)` | Sets the scalar field to be mapped and updates the color map (see below). |
| `showColorBar(flag)` | Shows/hides the side color bar. |
| `setColorTable(colors)` | Sets the color palette (N×3 matrix) used by the color map. |
| `setColorBarLimits(minv, maxv)` | Sets the min/max limits shown on the color bar. |

`setScalars(field, varargin)` forwards its arguments to `ScalarExtractor.setField`:

- `field` can be a custom `ScalarField` object, or the string `'u'` (displacement) or `'t'` (traction);
- when `field` is `'u'` or `'t'`, `varargin` (optional) selects how the scalar is extracted from the vector at each tessellation point `P` of an element:
  - `'direction', v` — projection onto the fixed direction `v` (a 3D vector);
  - `'normal'` — projection onto the local surface normal at `P`;
  - a dof spec, given as a char array combining `'x'`, `'y'`, `'z'` (e.g. `'z'`, `'xy'`, `'xyz'`), or the equivalent numeric array (`1`, `2`, `3`) — the scalar is the corresponding component, or the magnitude of the vector restricted to the given components, of the displacement/traction at `P`;
  - if omitted, defaults to dof `'z'`.

```matlab
mi.setScalars('u', 'z');      % z-displacement
mi.setScalars('t', 'normal'); % traction projected onto the surface normal
mi.showColorMap(true);
mi.showColorBar(true);
```

### 5.6 Deformed / undeformed mesh and remeshing

| Command | Description |
|---|---|
| `deformMesh(scale)` | Applies mesh deformation with factor `scale` (`0` removes the deformation, restoring the original geometry). While deformed, an overlay copy of the undeformed mesh is kept. |
| `showUndeformedMesh(flag)` | Shows/hides the undeformed-mesh overlay. |
| `setUndeformedMeshAlpha(value)` | Sets the opacity (0–1) of the undeformed-mesh overlay. |
| `setUndeformedMeshColor(value)` | Sets the color of the undeformed-mesh overlay. |
| `remesh(resolution)` | Retessellates the mesh with a new resolution; recomputes `bounds` and redraws, preserving the color map if it was active. |
| `flipVertexNormals(flag, sid)` | Flips (`true`) / restores (`false`) the vertex normals of a shell (`sid`), or of the currently selected elements if `sid` is omitted. |
| `meshColoring(colorMapHandler, shuffle)` | Colors elements by graph-coloring of the mesh, using a colormap handler (default `@jet`), shuffling colors by default (`shuffle`, default `true`). |

```matlab
mi.deformMesh(5);            % apply deformation with scale 5
mi.showUndeformedMesh(true);
mi.remesh(8);
```

### 5.7 Vectors (displacements and tractions)

| Command | Returns | Description |
|---|---|---|
| `showVectors(flag, eids)` | — | Shows/hides vectors on elements. `flag` can be `true`/`false`, or the strings `'u'` (displacement), `'t'` (traction), or `'update'` to refresh the current vector handler. `eids` restricts the display to specific elements. |
| `showVectorsByRegion(handle, colors)` | `nr` | Draws vectors per mesh region, each region in a different color (default: `axes.ColorOrder`). `handle` can be `'u'`, `'t'`, or a custom function handle. |
| `setVectorScale(scale)` | — | Sets the vector scale factor (`0` disables auto-scale). |
| `setVectorColor(color)` | — | Sets the vector color. |

```matlab
mi.showVectors('u');          % displacement vectors on all elements
mi.showVectors('t', [1 2 3]); % traction vectors on elements 1, 2 and 3
mi.showVectorsByRegion('u');
```

### 5.8 Load points and ground plane

| Command | Description |
|---|---|
| `showLoadPoints(flag)` | Shows/hides the load points associated with the mesh nodes. |
| `showGround(flag)` | Shows/hides the reference ground plane. |

```matlab
mi.showLoadPoints(true);
mi.showGround(true);
mi.groundFaceColor = [0.6 0.6 0.9];
```

### 5.9 Constraints and loads

| Command | Returns | Description |
|---|---|---|
| `makeConstraint(dofs, evaluator, ...)` | `c` | Creates a constraint (`ConstraintGroup`) on degrees of freedom `dofs` of the currently selected elements, computing prescribed displacement values with `evaluator`. |
| `deleteConstraint(constraint)` | `b` (logical) | Removes one or more constraints. |
| `deleteAllConstraints()` | `b` (logical) | Removes every constraint from the mesh. |
| `makeLoad(evaluator, ...)` | `l` | Creates a load (`LoadGroup`) on the currently selected elements, computing prescribed traction values with `evaluator`. |
| `deleteLoad(load)` | `b` (logical) | Removes one or more loads. |
| `deleteAllLoads()` | `b` (logical) | Removes every load from the mesh. |

`makeConstraint` and `makeLoad` are not entirely symmetric: only `makeConstraint` takes a `dofs` argument, and `makeLoad` recognizes two extra string shortcuts (`'pressure'`, `'torque'`) that `makeConstraint` does not. Each is detailed below.

#### `dofs` (`makeConstraint` only)

`dofs` selects which displacement components are prescribed. It can be given as:

- a char array combining `'x'`, `'y'`, `'z'` (e.g. `'x'`, `'xz'`, `'xyz'`), or
- the equivalent numeric array with values in `{1, 2, 3}`.

A `Load`, in contrast, always affects all three traction components at once — it has no `dofs` argument, and `makeLoad` takes `evaluator` as its first parameter.

#### `evaluator` — common forms (both `makeConstraint` and `makeLoad`)

Whenever `evaluator` is **not** a char array, both commands parse it the same way (`BC.parseArgs`/`BCFunction.parse`, see [`BC.m`](../bea/BC.m)/[`BCFunction.m`](../bea/BCFunction.m)):

- a numeric constant (scalar, applied to every prescribed dof and every node), or a numeric vector with one value per dof (`makeConstraint`) / per component (`makeLoad`);
- a function handle with signature `value = evaluator(u, v, P)`, where `u, v` are the local parametric coordinates (in `[-1, 1]`) of the evaluation point on the element and `P` is its position (3D) in global coordinates; it is evaluated once per element node.

An optional trailing argument then sets the direction along which a scalar `evaluator` value is applied:

- `'direction', v` — a fixed 3D direction `v`;
- `'direction', 'normal'` — the local surface normal at each evaluation point;
- `'direction', 'none'` (default) — no direction; each numeric component of `evaluator` maps directly to the corresponding dof/component.

#### `evaluator` — `makeConstraint`-only string shortcuts

For `makeConstraint`, `evaluator` may also be one of these strings, handled by the generic `BCFunction.parse`:

- `'constant'` followed by a scalar value `z`: builds a function that returns `z` regardless of `u`, `v`, `P` (equivalent to `BCFunction.constant(z)`);
- `'bilinear'` followed by `z1, z2`: builds a bilinear function of the local coordinates, `f(u, v) = a(u) * b(v)`, where `a` linearly interpolates `z1` and `b` linearly interpolates `z2` over `[-1, 1]`. Each of `z1`, `z2` can be a scalar (constant along that direction) or a 2-element vector `[value_at_-1, value_at_+1]`.

**These two shortcuts are not available for `makeLoad`.** `Load` overrides the argument parsing (`Load.parseArgs`, see [`Load.m`](../bea/Load.m)) and intercepts *any* char `evaluator` itself, recognizing only `'pressure'` and `'torque'` (below); passing `'constant'`/`'bilinear'` to `makeLoad` does not fall back to the generic parsing and results in an error.

#### `evaluator` — `makeLoad`-only string shortcuts

For `makeLoad`, `evaluator` may instead be one of these strings (handled by `Load.parseArgs`, see [`Load.m`](../bea/Load.m)):

- `'pressure', p` — shorthand for a scalar pressure `p` applied along the local surface normal; equivalent to calling `makeLoad(p, 'direction', 'normal')`;
- `'torque', O, D, S` — a torque load about the axis passing through point `O` (3D) with direction `D` (3D, normalized internally) and scale `S`. Internally builds the load field `F(P) = S * (D × r)`, where `r` is the component of `P - O` perpendicular to the axis (so that the torque about the axis equals `S * |r|`, in the direction given by `D`). Always applied along all three components, regardless of any `'direction'` argument (which is not accepted together with `'torque'`).

```matlab
mi.selectElements([1 2 3]);

% Constraints
c1 = mi.makeConstraint('xyz', [0 0 0]);                      % clamp x, y, z to zero
c2 = mi.makeConstraint('xyz', -0.01, 'direction', 'normal'); % uniform inward normal displacement
c3 = mi.makeConstraint('z', 'bilinear', [0 1], 1);           % z ramps from 0 to 1 along u, constant along v

% Loads
l1 = mi.makeLoad([0 0 -1]);                                  % uniform load in -z
l2 = mi.makeLoad(-1000, 'direction', 'normal');              % pressure of -1000 along the local normal
l3 = mi.makeLoad('pressure', -1000);                         % equivalent shorthand for l2
l4 = mi.makeLoad('torque', [0 0 0], [0 0 1], 20);            % torque of magnitude 20 about the z axis through the origin
```
